Agreed inre the calling interface.

The library should have one main entry point `(source: Iterator[str]) -> IRResult`.
It should also have three low-level entry points for lexing, parsing, and IR emitting:
* `(source: Iterator[str]) -> Iterator[Token]`
* `(tokens: Iterator[Token]) -> ASTResult`
* `(ast: ASTResult) -> IRResult`
The exact definition of `Token` (or whatever you want to call it) doesn't necessarily need to be specified, but the fact that there is a `Token` (or whatever) type does need to be part of the interface definition, just as BaseResult, ASTResult, and IRResult are.  In addition, it should be defined that the `Token`, `ASTResult`, and `IRResult` types need to implement `__str__` and `__repr__` to allow a caller to display them to the user in either a general or debugging format.
